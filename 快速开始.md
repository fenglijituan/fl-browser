## 环境准备

### 1. 安装 Node.js

前往 https://nodejs.org/ 下载并安装 LTS 版本（推荐 v18+）

安装完成后，打开命令行验证：
```bash
node --version
npm --version
```

### 2. 安装 Git（可选，用于推送到 GitHub）

前往 https://git-scm.com/ 下载安装

## 本地运行

### 步骤 1：进入项目目录
```bash
cd "d:\网站HTML\FL Browser"
```

### 步骤 2：安装依赖
```bash
npm install
```

这会安装 Electron 和 electron-builder 等依赖包。

### 步骤 3：启动浏览器
```bash
npm start
```

浏览器窗口会打开，同时 DevTools 会开启用于调试。

## 打包 EXE 安装包

### 方法一：使用命令行（推荐）

#### 1. 打包便携版（ZIP）
```bash
npm run build:zip
```
输出文件：`dist/FL Browser-1.0.0-win.zip`
- 无需安装，解压即可使用
- 适合 U 盘携带

#### 2. 打包安装版（EXE）
```bash
npm run build:installer
```
输出文件：`dist/FL-Browser-Setup-1.0.0.exe`
- 标准 Windows 安装程序
- 可创建桌面快捷方式
- 可添加至开始菜单
- 可自定义安装路径

#### 3. 一键打包（双击运行）
```
一键打包.bat
```

### 方法二：使用打包脚本
```bash
node build-package.js
```

## GitHub 自动打包

### 1. 创建 GitHub 仓库

1. 登录 https://github.com
2. 点击右上角 **+** → **New repository**
3. 填写仓库名：`fl-browser`
4. 选择 **Public**（公开）
5. 点击 **Create repository**

### 2. 推送代码到 GitHub

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: FL Browser v1.0.0"

# 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/fl-browser.git

# 推送
git push -u origin main
```

### 3. 配置 GitHub Actions

代码推送后，GitHub Actions 会自动检测 `.github/workflows/build.yml` 文件。

### 4. 触发自动打包

有两种方式触发自动打包：

#### 方式 A：创建标签（推荐）
```bash
# 创建版本标签
git tag v1.0.0

# 推送标签到 GitHub
git push origin v1.0.0
```

GitHub 会自动开始打包，完成后在 Releases 页面发布。

#### 方式 B：手动触发
1. 前往 GitHub 仓库 → **Actions** 标签
2. 点击左侧 **Build FL Browser**
3. 点击右上角 **Run workflow**
4. 输入版本号，点击运行

### 5. 下载打包结果

打包完成后，有两种方式获取文件：

#### 方式 A：从 Actions 下载
1. 前往 **Actions** → 点击已完成的工作流
2. 在页面底部找到 **Artifacts**
3. 点击下载 ZIP 或 EXE

#### 方式 B：从 Releases 下载
1. 前往仓库 → **Releases**
2. 找到对应版本
3. 下载安装包

## 发布新版本

### 步骤 1：更新版本号
```bash
# 补丁版本（修复 bug）
npm version patch    # 1.0.0 -> 1.0.1

# 次版本（新增功能）
npm version minor    # 1.0.0 -> 1.1.0

# 主版本（重大更新）
npm version major    # 1.0.0 -> 2.0.0
```

### 步骤 2：提交更改
```bash
git add .
git commit -m "Bump version to v1.0.1"
git push
```

### 步骤 3：创建标签并推送
```bash
git tag v1.0.1
git push origin v1.0.1
```

### 步骤 4：等待自动打包
前往 GitHub Actions 页面查看进度。

## ⚙️ 配置选项

### 修改应用名称和图标

打开 `package.json`，修改以下字段：

```json
{
  "build": {
    "productName": "FL Browser",        // 应用显示名称
    "appId": "com.flbrowser.app",       // 应用唯一 ID
    "win": {
      "icon": "icons/wind-power-browser.ico"  // 图标路径
    }
  }
}
```

### 修改安装包选项

```json
{
  "build": {
    "nsis": {
      "oneClick": false,                    // false=显示安装向导
      "allowToChangeInstallationDirectory": true,  // 允许选择安装路径
      "createDesktopShortcut": true,        // 创建桌面快捷方式
      "createStartMenuShortcut": true,      // 创建开始菜单快捷方式
      "shortcutName": "FL Browser"          // 快捷方式名称
    }
  }
}
```

## ❓ 常见问题

### Q: npm install 很慢怎么办？
A: 使用淘宝镜像：
```bash
npm config set registry https://registry.npmmirror.com
```

### Q: 打包失败怎么办？
A: 检查：
1. Node.js 版本是否 >= 16
2. 磁盘空间是否足够
3. 网络连接是否正常

### Q: 如何修改版本号？
A: 编辑 `package.json` 中的 `version` 字段，或运行 `npm version x.x.x`

### Q: 打包后文件在哪？
A: 在 `dist/` 目录下
